The project version (v2.146.0) is strictly unified across the manifest, 'constants.ts', and all standalone scripts. To resolve discrepancies between 4-part manifest versions (e.g., 2.146.0.0) and 3-part semver used in scripts, the system employs a normalization utility in both the build pipeline ('check-version-sync.mjs') and the runtime version check hook. This logic converts version strings into a standard 3-part integer-based semver for comparison, preventing false 'version mismatch' UI banners while preserving human-readable 'version_name' for display. The build pipeline fails if version strings in metadata files like 'shared-state.ts' or 'instruction.ts' do not match the main manifest.
