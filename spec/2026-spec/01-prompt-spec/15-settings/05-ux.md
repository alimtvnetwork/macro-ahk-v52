# 05 — Settings UX

**Date:** 2026-06-02
**Task:** T95

## Layout

Single scrollable page. Each section is a card with:
- Section title + one-line description.
- Fields grouped logically (no more than 6 per card).
- Inline validation under each field.
- Per-section **Reset** button (right-aligned in card header).

## Save model

- **No autosave.** Explicit save per section (per the project's no-autosave preference for prompt editing — mirrors `08-save-create-edit/05-editor-ux.md`).
- Dirty indicator on the section header.
- `Ctrl+S` saves the focused section; `Ctrl+Shift+S` saves all dirty sections.

## Validation feedback

- Field-level on blur.
- Save aggregates errors into a single toast listing affected fields.
- Out-of-band warnings (e.g. delay < 5s) render as amber hints, not errors — save still proceeds.

## Accessibility

- Each section is a `<section aria-labelledby="...">`.
- Reset confirmations are modal dialogs with focus trap.
- All toggles expose `aria-checked`; sliders expose `aria-valuetext` with units.

## Empty / error states

- If a settings key is corrupt and reset to defaults, surface a one-time banner: *"<Section> settings were reset to defaults due to invalid stored data."* Dismissible.
